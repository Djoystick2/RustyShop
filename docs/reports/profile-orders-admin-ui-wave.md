# Profile Orders Admin UI Wave

Дата: 2026-03-13

## Что именно изменено

### Профиль

- `ProfilePage` пересобран в более цельную карточную структуру.
- Верхний блок профиля теперь оформлен как аккуратный hero с понятными meta-плашками.
- Пользовательский профиль получил:
  - более чистую композицию;
  - mini-card список избранного вместо простого списка ссылок;
  - более собранные секции про бренд, контакты и магазин.
- Админ-профиль получил:
  - более цельный admin hub;
  - выровненные быстрые действия;
  - более аккуратные сервисные и debug-секции;
  - более чистую интеграцию блока бренд-настроек.

### AdminPanel

- Форма бренд-настроек переработана в более понятный и чистый вид.
- Поля собраны в аккуратную сетку, добавлен helper-текст и выровнен action area.
- Loading-state кнопки сохранён.

### Admin-flow поверхности

- Обновлены стили для `admin-list`, `admin-item`, giveaway summary-блоков и связанных карточек.
- `GiveawayPage` логически не переписывался, но его текущие admin-секции получили более спокойную визуальную подачу через shared CSS.

## Что по Заказам

- В текущей кодовой базе отдельной сущности/страницы `Orders` не найдено.
- В `src/types/entities.ts` отсутствует order-модель, а поиск по `order/orders/заказы` не выявил самостоятельного пользовательского или админского order-screen.
- Поэтому эта волна покрыла реально существующие экраны: `ProfilePage`, `AdminPanel` и текущие admin-flow поверхности.

## Какие спорные места остались

- Отдельного экрана заказов в проекте сейчас нет, поэтому полноценная UI-волна по orders невозможна без появления соответствующего домена и экранов.
- Финальная оценка плотности обновлённого `ProfilePage` и giveaway-admin секций всё ещё требует живого Telegram Mini App smoke-test на реальном устройстве.
- В сборке по-прежнему участвует тяжёлый `header.png` из предыдущих шагов, поэтому perceived performance первого экрана стоит учитывать отдельно от этой UI-волны.
