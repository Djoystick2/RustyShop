# Product Card Open Modal

Дата: 2026-03-13

## Что изменено

- Карточки товара в homepage sections больше не ведут пользователя через кнопку `Открыть`.
- Для карточек главной страницы добавлен отдельный режим `onOpen`, в котором вся карточка становится tappable.
- По тапу на карточку товар теперь открывается прямо на месте в lightweight modal sheet без смены основного экрана и без перехода в `Каталог`.
- Для modal sheet добавлен новый компонент `src/components/products/ProductQuickViewModal.tsx`.
- Внутри quick view показаны:
  - изображение;
  - название;
  - цена;
  - статусные бейджи;
  - описание;
  - материалы, если они есть;
  - основные действия пользователя или администратора.
- Для обычного пользователя quick view сохраняет нужные действия:
  - покупка через существующий Telegram acquire flow;
  - переход к контактам продавца;
  - добавление в избранное;
  - опциональный переход в полный экран товара.
- Для admin quick view сохраняет базовые action-кнопки управления товаром и ссылку на полный экран.
- Route-based открытие товара в остальных местах приложения не ломалось:
  - `ProductCard` остался как был;
  - `ProductMiniCard` вне homepage режима продолжает работать по старому сценарию.

- `npm run typecheck`
- `npm run build`

Обе проверки завершились успешно.

## Спорные места

- Живой UX smoke-test в реальном Telegram Mini App не выполнялся из терминального окружения, поэтому взаимодействие sheet подтверждено кодом и сборкой, но не live-сценарием на устройстве.
- В quick view сохранена ссылка `Открыть полный экран`, потому что полный route-based сценарий товара в проекте всё ещё нужен в других частях интерфейса.
- Внутри модального окна не добавлялся сложный gallery/thumbnail режим, чтобы не утяжелять решение и не дублировать полноценную страницу товара.
