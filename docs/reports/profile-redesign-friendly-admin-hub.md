# Profile redesign: friendly admin hub

## Что переработано

- Полностью пересобран `ProfilePage` как продуктовый экран вместо технической панели.
- Убран доминирующий debug/runtime-текст с основной поверхности профиля.
- Техническая информация вынесена в нижний скрываемый блок `Диагностика`.
- Для локальной разработки `Developer tools` тоже убраны в отдельный collapsible-блок.
- Добавлен дружелюбный hero-блок с быстрыми действиями и компактной навигацией.
- Добавлен переключатель темы `system / light / dark` с сохранением выбора в `localStorage`.
- Для администратора собран понятный hub с отдельными зонами:
  - storefront / тексты;
  - товары / витрина;
  - розыгрыш;
  - проверка пользовательского сценария покупки.
- Для администратора оставлен пользовательский сценарий покупки через `ProductMiniCard` с `isAdmin={false}`, чтобы была доступна обычная кнопка покупки без смены роли.
- `AdminPanel` переработан в две компактные формы:
  - бренд и hero-тексты storefront;
  - тексты продавца, контакты и label кнопки покупки.
- Из `AppShell` убраны profile-specific service banners, из-за которых экран профиля выглядел как debug-console.

## Какие файлы изменены

- `src/pages/ProfilePage.tsx`
- `src/components/admin/AdminPanel.tsx`
- `src/components/layout/AppShell.tsx`
- `src/styles.css`
- `docs/reports/profile-redesign-friendly-admin-hub.md`

## Какие проверки выполнены

- `npm run typecheck`
- `npm run build`

## Спорные места и остаточные риски

- Явный выбор темы реализован на фронтенде через `data-theme` и `localStorage`; если позже понадобится серверная или Telegram-системная синхронизация темы, это будет отдельный проход.
- Визуальная часть подтверждена типизацией и production build, но не живым smoke-test внутри Telegram Mini App на устройстве.
- Блок `Диагностика` остаётся доступным для поддержки, но его содержимое всё ещё техническое по смыслу; это сознательно, чтобы не потерять служебную наблюдаемость.
