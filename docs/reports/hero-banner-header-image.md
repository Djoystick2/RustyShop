# Hero Banner Header Image

Дата: 2026-03-13

## Что именно изменено

- На главной странице в существующий верхний hero-секшен встроен отдельный banner-компонент с использованием файла `header.png` из корня проекта.
- Для баннера создан отдельный компонент `src/components/storefront/HeroBanner.tsx`.
- Баннер подключён в верхней части homepage hero-блока через `HomepageSectionRenderer`.
- Для баннера добавлена адаптивная посадка:
  - сохранён широкий banner-формат;
  - высота осталась умеренной;
  - изображение заполняет блок без ощущения громоздкости;
  - добавлены мягкие скругления и лёгкая внутренняя тень для интеграции в текущий стиль.
- Для тёмной темы добавлена отдельная лёгкая подложка, без тяжёлых эффектов и blur.

## Какие файлы изменены

- `src/components/storefront/HeroBanner.tsx`
- `src/components/storefront/HomepageSectionRenderer.tsx`
- `src/styles.css`
- `dist/index.html`
- `tsconfig.tsbuildinfo`

Примечание:

- `dist/index.html` обновился как артефакт production-сборки.
- `tsconfig.tsbuildinfo` обновился как артефакт TypeScript build.
- Файл `header.png` используется как исходный asset из корня проекта и остаётся отдельным файлом в репозитории.

## Какие проверки выполнены

- `npm run typecheck`
- `npm run build`

Обе проверки завершились успешно.

## Спорные места по мобильной посадке

- По композиции баннер посажен в широкий hero-формат и не должен съедать слишком много первого экрана, но финальную оценку обрезки персонажей и декоративных полок лучше подтвердить live-проверкой в Telegram Mini App на реальном устройстве.
- Сам файл `header.png` тяжёлый: production build показал итоговый asset около 1.17 MB. Логика и UI не страдают, но если потребуется дополнительная оптимизация первого экрана, следующий шаг — подготовить более лёгкую web-версию того же изображения без изменения композиции.
